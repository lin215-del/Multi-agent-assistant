# 演示指南 · 答辩现场操作手册

## 启动命令

```bash
# 1. 确认 .env 有 SILICONFLOW_API_BASE/KEY/RAGFLOW_BASE_URL/KEY/DATASET_ID
#    （参考 .env.example，缺的问助教/同学拿）

# 2. 激活 venv（项目根 .venv）
.venv\Scripts\activate     # Windows
# 或
source .venv/bin/activate  # macOS/Linux

# 3. 启动 FastAPI 后端（含前端静态托管）
python -m uvicorn web.backend.app:APP --host 127.0.0.1 --port 8090 --log-level info

# 4. 浏览器打开
#    http://localhost:8090
```

启动成功日志会显示 `Uvicorn running on http://127.0.0.1:8090` 和 `Application startup complete.`

## 演示流程（5 分钟版）

**0:00–0:30 开场**（念稿）：
> "暨大学生问答助手，基于 Docker+RAGFlow 的多智能体系统。今天演示第 8 步——前台展示层。"

**0:30–1:30 多模态差异化（**答辫主钩子**）**：
> "我们和别的组最大的区别在清洗层。别的组把 PDF 整本塞 RAGFlow——只能答'通知大概说了啥'。我们把表格单独转 JSON、图用 AI 生成文字描述，分别建索引。"
>
> 操作：输入第 5 题"期末考试什么时候"，展开"引用来源"。
>
> "看这条**表格**徽章——这是从 PDF 表格里抽出来的；这条**图**徽章——是 VLM 给流程图生成的文字描述。这条**正文**徽章——是普通段落。"

**1:30–2:30 智能体过程可见**：
> 操作：点"过程 trace"展开。
>
> "系统跑过 5 个智能体节点：调度判问题类型 → 检索拿资料 → 分析写答案 → 反思质检。Round 是反思重试次数——如果反思不通过，自动重写。这题 Round=0 是因为第一次答案就过了。"

**2:30–3:30 四路由演示**：
> 操作：依次点 1 / 3 / 4 / 7 题。
>
> "第一题只查资料——retrieve；第三题纯算——tool，结果带了 📊 工具计算框；第四题又查又算——both；第七题闲聊——reject，淡红横幅加'超出服务范围'。"

**3:30–4:30 历史可观测性**：
> 操作：切到"历史"tab。
>
> "每次问答都落 SQLite。展开看完整 trace——包括反思前的草稿。这就是 ABSD 质量属性里的'可观测性'：每个节点输出都能追。"

**4:30–5:00 收尾**：
> "整套系统跑通：清洗层 + 知识库层 + 多智能体层 + 应用层。第 8 步到此结束，谢谢老师。"

## 现场应急

| 问题 | 处理 |
|---|---|
| 端口被占 | 换 `--port 8091`，浏览器改地址 |
| LLM 超时 | 检查 .env 的 API key；SiliconFlow 国内服务绕 VPN |
| RAGFlow 502 | 检查 Docker：`docker ps \| grep ragflow`；重启 `docker compose up -d` |
| 前端样式错乱 | Ctrl+F5 强刷（CSS 缓存） |
| 答非所问 | 截屏给助教看；可能是 chunk 切分太碎，调 RAGFlow chunk_size |
| 表格/图徽章不显示 | 确认 query 命中 PDF 类文档；HTML 通知都是"正文"徽章（正常） |
| 忘开服务 | 评委给 2 分钟开；提前 30 分钟到答辩教室启动 |

  -H "Content-Type: application/json" \
  -d '{"question":"国奖申请条件是什么"}' | python -c "import sys,json; d=json.load(sys.stdin); print('route:',d.get('route'),'len:',len(d.get('answer','')))"
