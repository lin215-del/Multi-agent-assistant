## 多智能体过程可见（60 秒）

> （操作：展开底部的 trace 折叠区）

系统跑了 **5 个智能体节点**：
1. **调度**：判问题类型 → 走哪条路
2. **检索**：调 RAGFlow 拿 top-k 资料
3. **分析**：把资料拼成答案
4. **反思**：质检员检查答案——有没有引用来源、有没有跑题、有没有瞎编
5. 如果反思不通过，**自动重写**，最多 2 次

> （指 trace 上的 Route / Round / Latency / Reflection 四列）

Route 显示走的哪条路，Round 是反思轮数，Latency 是耗时，Reflection 是质检结论。这题 Round=0，是因为第一次答案就过了。

## ABSD 质量属性（60 秒）

> （操作：切到"历史"tab，展开一条历史）

我从架构角度讲两个属性：

**可观测性**：每次问答都落 SQLite。展开历史项，能看到完整 trace——包括反思前的草稿、被否决的版本。这对调试和答辩都重要——任何问题都能追到节点。

**可修改性**：模型/RAGFlow/单个 Agent 都能替换。比如想换 LLM，改一个环境变量就行。这正是我们没用 LangGraph 自带 RAG 的原因——那种反例把所有逻辑耦合在一起，换不了。

## 评委可能追问（准备）

**问：反思 Agent 用什么模型？**
答：32B（Qwen2.5-32B-Instruct）。7B 做路由分类足够，但反思要细致判断"答案是否胡编"，32B 更稳。

**问：表格转 JSON 怎么做的？**
答：清洗层 MinerU 第一步识别出表格 caption/headers/rows，我们用 Python 第二步 caption+header+rows 转 JSON 结构化数据，再灌进 RAGFlow。RAGFlow 普通切块时这些 JSON 跟正文一起被索引。

**问：图描述准确率？**
答：用 Qwen3-VL（视觉大模型）生成。看引用区那条"图"徽章就是它生成的——把流程图每一步都描述出来。如果图太模糊或太小，会回退到"[图N] 占位 + 人工补"。

**问：为什么不接组员已有的网页？**
答：组员走 Flask + 单文件 HTML，我们走 FastAPI + SPA。两者接口格式不一致（他的是 `/api/ask`，我们是 `/api/chat` 等六端点）。统一需要再做适配层，但我们的服务清单（12 张卡片）和分组逻辑是新的。

**问：生产部署怎么做？**
答：Docker compose 一键起 RAGFlow + FastAPI 两个容器；前端静态文件挂 Nginx 反向代理。数据落 Postgres 替换 SQLite（生产环境）。模型走 SiliconFlow API，不本地部署（推理硬件成本高）。

**问：为什么选 LangGraph 而不是别的编排框架？**
答：课件指定。LangGraph 的 StateGraph / Conditional Edge / Checkpoint 三个概念正好对应"多 Agent 共享状态 / 条件分支 / 反思重试"，其他框架要么不够灵活要么太重。

**问：拒答机制会不会让用户失望？**
答：会，但比胡编好。答辫演示里专门演示了"今天天气"拒答——这是可靠性 ABSD 属性的体现。生产环境可以加 fallback："拒答 + 推荐人工咨询链接"。
