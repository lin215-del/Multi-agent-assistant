暨大学生问答助手

基于 Docker + RAGFlow 的多智能体系统 · 实训方案 3（基于 Python 的 AI 应用开发）
暨南大学 2023 级本科毕业实习实训 · 计算机科学系 · 2026 年暑期

一、项目简介

面向暨南大学本科生的教务办事智能问答助手。把教务处（jwc）公开的通知、办事指南、规章制度等文档，经多模态清洗后灌入 RAGFlow 知识库；学生用自然语言提问，系统经多智能体检索 + 二次包装，返回带来源引用的答案，检索不到时拒答而非胡编。

解决的核心痛点：教务信息分散在各种通知/规章/PDF 里，学生找答案费时；本系统把它们汇聚成一个"问得到、答得准、查得到出处"的助手。

核心功能（MVP — 答辩主干）

#功能说明

MVP知识库jwc 公开文档清洗后入 RAGFlow，支持分块/向量/检索

三、系统架构（四层）

④ 应用层
用户提问 → 接口（FastAPI）→ 答案 + 溯源链接展示（Streamlit/Gradio 前台）
▲
③ 多智能体层（LangGraph）
调度 / 检索 / 分析 / 工具 / 反思 —— 任务规划、检索、调用工具、结果生成、错误检查
▲
② 知识库层（RAGFlow）
上传资料 → 解析分块 → 向量化（bge-m3）→ 检索（top-k）
▲
① 数据清洗层
爬虫采集 → HTML/PDF 多模态清洗（MineRU + Python，表转 JSON、可溯源）→ 入库

数据清洗是地基——课件原话："清洗不好，即便 RAGFlow 分块参数再好、多智能体再强，检索效果也会很差劲。"

四、数据流

# 离线：数据采集 + 清洗 + 入库 jwc 公开栏目（通知/办事指南/规章） │ crawl_jwc.py（爬虫，静态栏目自动采） ▼ raw/（HTML + PDF 附件） │ clean_html.py（HTML→markdown） │ clean_pdf.py（PDF→MineRU→markdown + 表→JSON） ← 进行中 ▼ cleaned/（干净 markdown + 表格 JSON，带溯源 URL） │ 灌入 RAGFlow 知识库 ▼ RAGFlow（分块 / bge-m3 向量 / 检索） # 在线：用户问答 学生提问 │ FastAPI ▼ 多智能体（LangGraph）：调度→检索→分析→反思 │ │ │ └─ 调 RAGFlow 拿 top-k → qwen2.5-7b 二次包装 ▼ 答案 + 来源引用（可点开核对）/ 检索不到则拒答

五、多智能体设计（LangGraph 编排）

调度 Agent
判断问题类型、路由到后续节点、控制停止条件
检索 Agent
调用 RAGFlow 查资料，拿 top-k 分块
分析 Agent
基于检索结果生成答案（qwen2.5-7b）
工具 Agent
调用 OCR / 计算器等外部工具
反思 Agent
检查答案可靠性、是否幻觉、要不要重试

LangGraph 核心概念：State（多 Agent 共享状态）/ Node（节点）/ Edge（执行顺序）/ Conditional Edge（条件分支）/ Checkpoint（断点恢复）。

八、ABSD 质量属性（答辩要点）

属性本项目怎么体现
性能Agent 调用链不能太长；RAG 检索 + Rerank 控制延迟；top-k 数量权衡
可靠性某 Agent 失败兜底；RAGFlow 没结果时降级 / 拒答
安全性Agent 工具权限控制；API Key / 用户数据保护
可修改性模型 / RAGFlow / 单个 Agent 可替换（不用方案 4 那种 LangGraph 自带 RAG 反例）
可观测性每个 Agent 的输入输出可追踪、错误能定位到节点
